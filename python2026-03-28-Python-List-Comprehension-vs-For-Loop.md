## Context
Python에서 데이터를 처리할 때, 가독성과 성능 사이에는 상충 관계가 존재합니다. 특히 리스트 컴프리헨션(List Comprehension)과 명시적인 `for` 루프 사용을 결정할 때 이러한 고려가 필요합니다. 이 문서는 주어진 상황에 따라 어떤 방식을 선택해야 하는지에 대한 실무 가이드라인을 제시합니다.

## Core
### 1. 데이터량이 적고, 협업/유지보수가 중요할 때 (가독성 우선)
프로젝트의 규모가 작거나, 비전공자 및 입문자 등 다양한 팀원들과의 협업 및 장기적인 유지보수성이 중요할 때는 코드의 가독성을 최우선으로 고려해야 합니다.

*   **복잡한 조건은 명확한 이름의 함수로 분리**: 조건이 복잡할 경우, 해당 조건을 판별하는 로직을 별도의 함수로 정의하여 코드의 역할을 명확히 합니다. 이는 코드 블록의 길이를 줄이고, 각 부분이 어떤 역할을 하는지 쉽게 이해할 수 있도록 돕습니다.
*   **명시적인 `for` 루프 또는 `filter()` 적극 활용**: 데이터 처리 로직을 직관적으로 이해할 수 있는 `for` 루프나 `filter()` 함수를 사용하여 코드의 흐름을 명확하게 표현합니다.

**예시:**
```python
def is_valid_user(user):
    return user.is_active and user.has_permission('admin')

users_data = [...] # 가정된 사용자 데이터
valid_users = []
for user in users_data:
    if is_valid_user(user):
        valid_users.append(user)

# 또는 filter 사용
valid_users = list(filter(is_valid_user, users_data))
```

### 2. 수십만 건 이상의 대량 데이터를 빠르게 처리해야 할 때 (성능 우선)
대량의 데이터를 고속으로 처리해야 하는 환경에서는 성능이 중요한 고려사항입니다. 이 경우, 리스트 컴프리헨션이 `for` 루프에 비해 효율적일 수 있습니다.

*   **리스트 컴프리헨션 사용**: Python의 리스트 컴프리헨션은 내부적으로 최적화되어 있어, 대규모 데이터셋 처리 시 `for` 루프보다 빠른 실행 속도를 제공합니다.

**예시:**
```python
data = range(1000000) # 가정된 대량 데이터
squared_numbers = [x * x for x in data]
```

*   **성능과 가독성을 동시에 챙기고 싶을 때**: 대량 데이터를 처리하면서도 일정 수준의 가독성을 유지하고 싶다면, 조건부 로직만 함수로 분리하여 리스트 컴프리헨션과 결합하는 방식을 추천합니다.

**예시:**
```python
def is_even(number):
    return number % 2 == 0

data = range(1000000) # 가정된 대량 데이터
even_numbers = [x for x in data if is_even(x)]
```

## Insight
Python에서 데이터를 처리하는 방식은 프로젝트의 요구사항에 따라 유연하게 선택되어야 합니다. 가독성과 유지보수성이 중요한 상황에서는 명시적인 `for` 루프나 `filter()`와 함께 함수 분리를 통해 코드의 이해도를 높이는 것이 좋습니다. 반면, 수십만 건 이상의 대량 데이터를 처리하는 성능이 critical한 경우에는 리스트 컴프리헨션을 활용하여 효율성을 극대화해야 합니다. 두 가지 요구사항을 모두 충족해야 할 때는 조건 로직을 함수로 분리한 리스트 컴프리헨션이 좋은 대안이 될 수 있습니다. 핵심은 프로젝트의 우선순위를 명확히 하고, 그에 맞는 최적의 방법을 선택하는 것입니다.
