# 파이썬 리스트 컴프리헨션: 가독성 vs. 성능 고려 가이드라인

## Context

파이썬에서 리스트를 생성하거나 변형할 때 `리스트 컴프리헨션(List Comprehension)`은 매우 강력하고 간결한 문법을 제공합니다. 그러나 여러 조건을 포함하거나 복잡한 로직을 적용할 경우 가독성이 저하될 수 있다는 단점이 있습니다. 특히 `filter()` 함수와 비교했을 때, 가독성과 성능 사이의 균형점을 어떻게 찾아야 할지에 대한 고민이 필요합니다.

## Core

리스트 컴프리헨션 내부에 여러 개의 `if` 조건을 연달아 사용하면 (예: `[x for x in my_list if condition_a if condition_b]`) 코드가 간결해 보일 수 있지만, 조건을 한눈에 파악하기 어려워 가독성이 크게 떨어집니다.

**예시: 가독성이 떨어지는 리스트 컴프리헨션**

```python
data = range(1, 21)
filtered_data = [x for x in data if x % 2 == 0 if x > 10]
print(filtered_data) # [12, 14, 16, 18, 20]
```

이러한 경우, `filter()` 함수를 사용하는 것이 가독성 측면에서 더 유리할 수 있습니다. 특히 비전공자나 초보 개발자에게는 명시적인 함수 사용이 코드의 흐름을 이해하는 데 도움이 됩니다.

**예시: `filter()`를 활용한 가독성 개선**

```python
data = range(1, 21)

def is_even_and_greater_than_ten(x):
    return x % 2 == 0 and x > 10

filtered_data_with_filter = list(filter(is_even_and_greater_than_ten, data))
print(filtered_data_with_filter) # [12, 14, 16, 18, 20]
```

하지만, 대량의 데이터를 처리할 때는 리스트 컴프리헨션이 `filter()`와 람다 함수 조합 또는 별도의 함수 정의보다 성능 면에서 우위를 보이는 경우가 많습니다. 이는 리스트 컴프리헨션이 C로 구현된 내부 반복자를 사용하기 때문에 파이썬 인터프리터 오버헤드가 적기 때문입니다.

## Insight

**가이드라인:**

1.  **가독성 우선 (비전공자, 초보자, 간단한 조건):** `filter()` 함수를 활용하여 코드를 명확하게 작성합니다. 특히 여러 조건을 조합해야 할 경우 `and`, `or`를 사용하여 하나의 함수 내에서 조건을 명시하는 것이 좋습니다.
2.  **성능 우선 (대량 데이터, 성능 Critical):** 리스트 컴프리헨션을 사용합니다. 가독성을 해치지 않는 선에서 `if` 조건을 결합하거나, 복잡한 조건은 외부 함수로 분리하여 컴프리헨션 내에서는 최소한의 로직만 유지하는 방법을 고려할 수 있습니다.

결론적으로, 개발 목적과 대상 독자를 고려하여 리스트 컴프리헨션과 `filter()` 중 적절한 방법을 선택하는 유연한 접근 방식이 중요합니다. 일반적으로 리스트 컴프리헨션이 파이써닉하고 성능상 이점이 있지만, 가독성이 현저히 떨어지는 경우 `filter()`가 더 나은 대안이 될 수 있습니다.
